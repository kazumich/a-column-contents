## Claude Code とは？

Anthropic が提供する **Claude Code** は、ターミナル上で動作するAIコーディングアシスタントです。チャット形式でコードの生成・編集・デバッグを依頼できるだけでなく、プロジェクトのファイルを直接読み書きしてくれるのが最大の特徴です。

「AIにコードを書いてもらう」というと、ChatGPTやGitHub Copilotを思い浮かべる方も多いと思いますが、Claude Code はより**エージェント的**な動作をします。つまり、「このバグを直して」と言えば、自分でファイルを探し、原因を分析し、修正まで一気にやってくれるイメージです。

筆者はプログラミング歴1年ほどの初心者ですが、実際に使ってみてかなり感動したので、その体験を共有したいと思います。

## インストールと初期設定

### 必要なもの

- Node.js（v18以上）
- Anthropic の APIキー（[公式サイト](https://www.anthropic.com/)で取得）

### インストール手順

まずは npm でグローバルインストールします。

```bash
npm install -g @anthropic-ai/claude-code
```

次に、APIキーを環境変数に設定します。

```bash
export ANTHROPIC_API_KEY="your-api-key-here"
```

毎回設定するのが面倒な場合は、`~/.zshrc` や `~/.bashrc` に追記しておくと便利です。

### 起動する

プロジェクトのルートディレクトリに移動して、以下のコマンドを実行するだけです。

```bash
claude
```

これだけでClaude Codeが起動し、対話形式でやり取りができるようになります。シンプルで驚きました。

## 実際に使ってみた：3つのシーン

### シーン1：バグ修正を任せてみた

Pythonで書いた簡単なWebスクレイパーが動かず困っていたので、Claude Codeに丸投げしてみました。

```
> このプロジェクトのスクレイパーが動かないので原因を調べて直してください
```

すると Claude Code は自動的に関連ファイルを開き、エラーの原因（ライブラリのバージョン不一致）を特定して、`requirements.txt` と該当コードを修正してくれました。自分でStack Overflowを検索していたら30分はかかっていた作業が、**約2分で完了**しました。

### シーン2：READMEを自動生成してもらった

「READMEを書くのが苦手」というのは多くの初心者が感じることだと思います。Claude Codeにお願いしてみたところ、プロジェクトのコードを自動で読み込んで、セットアップ方法・使い方・ライセンス情報まで含んだ丁寧なREADMEを生成してくれました。

```
> このプロジェクトのREADME.mdを日本語で作成してください
```

生成されたドキュメントのクオリティは十分実用的で、そのまま使えるレベルでした。

### シーン3：コードのリファクタリング

「なんとなく動くけど汚いコード」をClaudeに見せてリファクタリングをお願いしました。関数の分割、変数名の改善、コメントの追加まで行ってくれ、コードレビューを受けているような感覚でした。さらに「なぜこの書き方の方が良いのか」を説明してくれるので、**学習ツールとしても優秀**です。

## 初心者が感じたメリット・デメリット

### メリット

- **自然な日本語で指示できる**：英語が苦手でも安心して使えます
- **ファイルを自動で操作してくれる**：コピペの手間がなくなる
- **説明が丁寧**：なぜそのコードにしたかを教えてくれるので勉強になる
- **コンテキストを保持してくれる**：会話の流れを覚えているのでやり取りがスムーズ

### デメリット

- **APIの利用料がかかる**：使いすぎると費用が膨らむので注意が必要
- **大規模プロジェクトでは遅くなることがある**：ファイル数が多いと応答に時間がかかる場合も
- **100%信頼するのは危険**：生成されたコードは必ず自分の目で確認する習慣をつけましょう

---

## 初心者へのアドバイス

Claude Code を使う上で意識したいポイントをいくつか挙げておきます。

1. **指示は具体的に書く**：「直して」より「〇〇のエラーが出るので原因を調べて修正して」の方が精度が上がります
2. **変更前にgit commitしておく**：ファイルを直接編集するので、元に戻せるよう必ずバックアップを
3. **小さなタスクから始める**：最初は1ファイルの修正など、スコープの小さい依頼から慣れていくのがおすすめ
4. **生成コードを読む習慣をつける**：ブラックボックスにせず、何が変わったかを必ず確認しましょう
